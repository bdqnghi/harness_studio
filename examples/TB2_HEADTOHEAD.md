# harness_studio vs AHE on Terminal-Bench 2 — runbook

A controlled, **same-input-harness** head-to-head: does harness_studio's optimizer
produce a better TB2 harness than agentic-harness-engineering (AHE) from the
**identical** starting harness, actor model, scorer, and task pool?

* **Input harness (both arms):** AHE's `agents/code_agent_simple/` — a bare nexau
  agent (one `run_shell_command` tool, ~9-line prompt, empty memory).
* **Scorer (both arms):** `harbor run --agent nexau --ak config_path=<ws>/code_agent.yaml -e docker`.
* **Actor (frozen, both arms):** `gpt-5.4` (an `${env.LLM_MODEL}` reference, so no
  edit can change it). Optimizer AI helpers use `claude -p` (ours) / gpt-5.4 (AHE).
* **Metric:** pass-rate on a **locked held-out pile** of 6 tasks (`tb2_config.py`,
  seed=7), scored for baseline / AHE-best / ours-best. Also report `cost_per_point`.

AHE's paper 69.7%→77.0% is unreproduced prose (its own work-report says full scale
is infeasible on a laptop); this is a fair method-vs-method run, not a paper repro.

## 0. Prerequisites (the strong machine)

* **Docker** with a generous VM (≥ 24 GB RAM, ≥ 8 CPUs) — each task uses ~2 GB, so
  RAM caps `--n-concurrent`. (This dev laptop had only 7.65 GB → n_concurrent ≤ 3.)
* **uv**, **Python 3.13**, the **`claude`** CLI (authed), and **gpt-5.4 creds**.
* Both repos checked out: `harness_studio/` and `agentic-harness-engineering/`.

```bash
# AHE: installs its pinned harbor (registers the `nexau` agent) + nexau v0.3.9
cd agentic-harness-engineering && uv sync

# harness_studio

# point the adapter at this machine's paths (defaults assume the dev laptop)
export AHE_DIR=/abs/path/to/agentic-harness-engineering
```

Sanity (no spend): `python examples/run_nexau_tb2.py --dry-run` prints the split +
the exact harbor command; `python examples/tb2_config.py` prints the locked split.

## 3. The full head-to-head (the real run)

```bash
# baseline on the locked held-out 6
python examples/tb2_score.py "$AHE_DIR/agents/code_agent_simple" --label baseline --k 3 --n-concurrent 8

# --- AHE arm ---
python examples/tb2_ahe_arm.py prepare --iterations 8 --n-concurrent 8
cd "$AHE_DIR" && uv run python evolve.py --config configs/experiments/exp-tb2-h2h.yaml
cd -                                   # note the experiments/<ts>__tb2-h2h dir it created
python examples/tb2_ahe_arm.py extract-best "$AHE_DIR/experiments/<ts>__tb2-h2h" /tmp/ahe_best
python examples/tb2_score.py /tmp/ahe_best --label ahe --k 3 --n-concurrent 8

# --- harness_studio arm ---  (uses the locked 16-task set + held-out 6 from tb2_config)
python examples/run_nexau_tb2.py --rounds 6 --segment-length 2 --strategies 3 \
  --n-concurrent 8 --proposer-model claude-opus-4-8     # opus = stronger proposer (recommended)

# --- verdict ---
python examples/tb2_compare.py
```

For the headline number, re-score each arm's best harness on the held-out 6 at
`--k 3` (or several `run_idx`) to cut binary variance — the gate already used
noise-aware acceptance, but the final report should average multiple rollouts.

## Fairness design (what makes the comparison honest)

* **Same everything but the optimizer:** identical input harness, actor model
  (gpt-5.4, env-locked), harbor scorer, and optimization pool (the 10 non-held-out
  tasks). The held-out 6 are never used for optimization by either arm.
* **Equal editing freedom:** harness_studio's part map (`run_nexau_tb2.nexau_part_map`)
  lets the Strategist ADD new tools/middleware/skills and register them in
  `code_agent.yaml` — exactly AHE's degrees of freedom. Only the frozen `llm_config`
  and inert infra files are off-limits (Strategist skill + env model reference).
* **Disclosed deviations (AHE arm):** Agent Debugger OFF and explore_agent OFF
  (no SERPER key; the only locally-validated AHE path), evolve effort = `high`
  (not `xhigh`). Re-enable for a maximal-fidelity AHE run if desired.

## Tuning / levers (if our arm doesn't win the first run)

* `--proposer-model claude-opus-4-8` (stronger edits; the default Tier-A is sonnet).
* More `--rounds` / `--strategies`; `--segment-length < rounds` so the meta-loop fires.
* Feed real harbor failure transcripts to the Diagnoser/Strategist (not yet wired) —
  the highest-leverage future enhancement, mirroring AHE's trace/ADB analysis.
* Our structural edge over AHE's greedy loop: the noise-aware gate + deep-audit
  rewind never bank a regression — most valuable when raising `--k` at the gate.
